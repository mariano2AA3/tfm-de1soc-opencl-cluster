Description

This benchmark demonstrates a OpenCL implementation of the Lucas Kanade Optical Flow algorithm. The implementation is based on the following paper: Pyramidal Implementation of the Lucas Kanade Feature Tracker Description of the algorithm, by Jean-Yves Bouguet.

The current version is a dense, non-iterative, non-pyramidal version with 52x52 window size. Presentation shows high-level details of implementation.

The input data is is located in bin directory. The actual input format used by the host program is .raw, which is just a space-separated dump grayscale values for each pixel. Bitmap (.bmp) files for the input are provided for reference. Output is in PPM format.

Compiling the OpenCL Kernel

The top-level OpenCL kernel file is device/optical_flow.cl.

To compile the OpenCL kernel, run:

aoc device/optical_flow.cl -o bin/optical_flow.aocx --board <board>

where <board> matches the board you want to target. The -o bin/optical_flow.aocx argument is used to place the compiled binary in the location that the host program expects.

If you are unsure of the boards available, use the following command to list available boards:

aoc --list-boards

Kernel Preprocessor Definitions

The kernel has the following preprocessor definitions:

Define Type Default Description
-DWINDOW_SIZE=<#> Optional 56 Window size.

Compiling the Host Program

To compile the host program, run:

make

The compiled host program will be located at bin/host.

Running the Host Program

Before running the host program, you should have compiled the OpenCL kernel and the host program. Refer to the above sections if you have not completed those steps.

To run the host program on hardware, execute:

bin/host

Host Parameters

The general command-line for the host program is:

bin/host [-iters=<#>]

where the one parameter is:

Parameter Type Default Description
-iters=<#> Optional 100 Number of iterations to run.

OpenCL Binary Selection

The host program requires a OpenCL binary (AOCX) file to run. For this example design, OpenCL binary files should be placed in the bin directory.

By default, the host program will look for a binary file in the following order (earlier pattern matches take priority):

  1. A file named optical_flow.aocx.
  2. A file named optical_flow_<board>_161.aocx, where <board> is the name of the board (as passed as the --board argument to aoc).
